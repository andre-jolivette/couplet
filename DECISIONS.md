# Couplet — Architectural Decisions & Current State

## Project Brief
Couplet is a macOS app (bundle ID `com.toastbrigade.Couplet`, deployment target macOS 14.0) for street and documentary photographers. It discovers meaningful image pairs in a photo library — not duplicates or sequential shots, but conceptually resonant connections the photographer might not have noticed. The serendipity machine goal: surface pairs that reward a second look.

## Engine Architecture

### Indexing Pipeline (5 phases)
1. **Scan** — FileScanner reads EXIF captureDate, colorProfile (color/bw) from CGImageSource
2. **Duplicate detection** — dHash perceptual hashing, Hamming threshold=6, hero image selection
3. **Thumbnails** — CGImageSourceCreateThumbnailAtIndex at 512px max (prevents IOSurface exhaustion)
4. **CLIP extraction** — CLIPCoreMLEngine, 224px input, cosine similarity embeddings
5. **Captioning** — OllamaCaptioningEngine → moondream via localhost:11434. Captions ALL uncaptioned active images in DB (not just current scan batch). Clears malformed captions with label prefixes on each run.
6. **Pair scoring** — PairScorer with dual topK (composite + thematic), then DELETE+INSERT to replace stale pairs

### Scoring (PairScorer.swift)
- **Aesthetic** — HSL histogram intersection (harmony) vs LAB palette contrast
- **Geometric** — edge orientation cosine similarity + composition grid cosine similarity
- **Thematic** — ConceptClusters vocabulary matching on moondream captions; Dice coefficient on matched cluster sets; falls back to CLIP cosine similarity if no captions
- **Composite weights** — aesthetic×0.40 + geometric×0.20 + thematic×0.40 (thematic boosted to 0.60 when score ≥0.20)
- **Penalties** — sequential shots (≤30s: ×0.40, ≤60s: ×0.55, ≤5min: ×0.85), filename-base duplicates (zeroed), CLIP similarity ceiling >0.88 (×0.40), cluster saturation: weight-based gate (weightedSharedSum > 5.0 → zeroed; previously count > 3)
- **Dual topK** — top-50 by composite per image + top-10 by thematic score per image (minimum 0.20)

### ConceptClusters (ConceptClusters.swift)
29 semantic clusters: skilled_performance, sound_music, sensory_overwhelm, bodily_gesture, tenderness_care, isolation_solitude, community_gathering, tension_conflict, joy_celebration, grief_sorrow, labor_effort, movement_energy, stillness_rest, ritual_ceremony, urban_street, vulnerability_exposure, power_dominance, looking_watching, devotion_belief, confinement_freedom, youth_age, waiting_anticipation, transformation_change, nature_landscape, humor_absurdity, uncanny_ordinary, economic_precarity, solitude_in_crowd, domestic_intimacy

Tokenizer strips common suffixes (ation, tion, ance, ence, ing, ed, er, etc.) before matching.
Thematic score uses **Dice coefficient** (not Jaccard) — less harsh when one image has many clusters.
Two-signal clusters (require ≥1 keyword hit from each of two vocabulary groups): humor_absurdity, uncanny_ordinary, solitude_in_crowd, domestic_intimacy, economic_precarity.

### Database Schema (GRDB, SQLite WAL)
- `images` — path, contentHash, filename, folderID, captureDate (Double, Unix timestamp), colorProfile ("color"/"bw"), caption (TEXT), dHash, duplicateGroupID, isHero, isActive
- `pairs` — imageAID, imageBID, aestheticScore, geometricScore, thematicScore, compositeScore, rationale, rawEdgeSim, rawGridSim, maxEdgePeakedness, maxGridVariance, edgePeakednessMult, gridVarianceMult
- Migrations: v1_initial, v2_duplicates, v3_captions, v4_colorProfile, v5_geometricStats, v6_distinctivenessMultipliers

## App Architecture

### Key files
- `EngineController.swift` — @MainActor bridge. Builds CLIP + captioning engines in cancellable Task. fetchPairs: fetches 2000 from DB, applies per-image cap of 15, trims to 750
- `PairScorer` modality labeling — thematic if thematicScore ≥0.25 AND thematicScore > geometricScore; otherwise geometric if geometric ≥ aesthetic
- `LightboxInfoRail.swift` — right rail showing captions, cluster breakdown, score bars, composite breakdown. Opened by clicking score block in lightbox top bar

### UI State
- Grid: 750 pairs displayed, per-image cap of 15
- Filters: All/Aesthetic/Geometric/Thematic pills, Sort dropdown (Composite/Thematic/Geometric/Aesthetic), All tones/Color/B&W/Mixed picker, Show hidden toggle, Search
- Sort is display-time only — independent of modality filter pills; combine freely (e.g. filter to Thematic, sort by geometric score)
- Hide sequential pairs: moved to Settings (Display section), persisted via UserDefaults, synced into PairsGridViewModel at display time
- Lightbox: score block (A/G/T + composite) clickable → slides in info rail
- Score pills: opacity scales with score value; color-coded (blue=aesthetic, green=geometric, orange=thematic)

### Thematic scoring problems
- **Labeling bias**: geometric scores (0.80+) dominate because most street photography has strong structural similarity. Thematic scores (0.20–0.45) rarely beat geometric despite being meaningful. Needs Option B (store selection reason at index time) and Option D (improve geometric scorer)
- **Score ceiling**: confirmed Jaccard hard-caps at 0.60 given the asymmetry + saturation gates. Switched to Dice — new ceiling 0.75. Existing DB pairs retain old Jaccard scores until re-indexed.
- **Cluster weight equity**: all cluster matches currently count equally, but `urban_street` overlap is far less meaningful than `grief_sorrow` overlap. Weighted Dice is the next step (see backlog #17).
- **Vocabulary gaps**: clusters covering sound/music and sensory experience added; violin+ear-covering pair now scores thematic=0.25. More gaps likely exist
- **Caption quality**: moondream (1.8B) describes surface action accurately but misses subtext, irony, cultural references. Acceptable for cluster matching; not good enough for nuanced semantic understanding

1. **Option B** — store `selected_for` flag ('thematic'/'composite') in pairs table at scoring time; use for labeling instead of post-hoc score comparison
2. **Option C** — already done (three score pills + info rail)
3. **Option D** — improve geometric scorer to be more selective; currently rewards any two street photos with similar subject scale
4. **Color tone filter** — ✅ Fixed. Three-tier detection: Gray color model → ICC profile name → pixel-level chroma sampling (catches RGB B&W JPEG exports from Lightroom). Backfill runs for all active images on every re-index (same pattern as caption backfill), so a single re-index of any folder corrects the whole library.

## Ollama / Captioning Setup
- Install: `brew install ollama`, then `ollama pull qwen2.5vl:7b`
- Register the trimmed context variant: `ollama create qwen2.5vl-caption -f ConjunctEngine/Modelfile`
- Run: `ollama serve` (or `brew services start ollama` for auto-start)
- Current model: `qwen2.5vl-caption` — qwen2.5vl:7b with num_ctx capped at 2048 (see ConjunctEngine/Modelfile). Prompt is a single flowing paragraph instruction covering scene context, social situation, emotional register, symbolism, irony, and contrast. No structured scaffolding.
- Timeout: 120s per image; URLSession ceiling 150s. Actual caption time ~3–6s/image on M1 Max.
- num_predict: 400 tokens (raised from 250; see decisions log). At 250, 57% of captions ended mid-sentence (~1,100–1,200 chars), silently dropping the interpretive language qwen writes in the second half of its paragraph. At 400 tokens ≈ 1,600–2,000 chars, qwen completes naturally within the num_ctx 2048 budget (prompt ≈ 250 + image ≈ 250 + output 400 = ~900 total). Existing truncated captions require a full re-caption pass — re-score alone doesn't help.

## Decisions Log

| Decision | Rationale |
|----------|-----------|
| Security-scoped bookmark for CLIP model | App Sandbox requires it; model lives outside app bundle |
| 512px thumbnail for CLIP input | Full-res caused IOSurface exhaustion during concurrent extraction |
| Soft temporal falloff (not hard cliff) | Hard 30s cliff was too aggressive for street photography shot quickly |
| Dice coefficient for thematic | Jaccard penalised protest photos (6 clusters) pairing with portraits (2 clusters) too harshly. Dice penalises each unique cluster once (denominator only) vs. Jaccard's double-penalty (numerator + denominator). Both asymmetry gate (≥1 unique per side) and saturation gate (>3 shared → zero) are preserved. Ceiling: 3 shared + 1 unique each → 6/8 = 0.75, comparable to aesthetic (0.84) and geometric (0.79) maxima in the current library. |
| 2000 DB fetch → cap → 750 display | Per-image cap + thematic re-weighting shrank 750 fetch to ~380 displayed |
| Per-image cap of 15 | Car interior image had 117 pairs, crowding out rest of library |
| topK=50 + thematic topK=10 | Thematic pairs never made composite top-50; separate selection ensures they appear |
| DELETE+reINSERT pairs on re-index | record.save() hit UNIQUE constraint silently; old scores were never updated |
| Caption all active images (not just scan batch) | Images from previous scans weren't getting captioned on re-index |
| colorProfile: pixel-level chroma sampling threshold=15 | `kCGImagePropertyColorModel == "Gray"` misses B&W JPEG exports from Lightroom/Capture One (stored as RGB, zero chroma). 32×32 downsample + max per-channel diff ≤ 15/255 catches these while absorbing JPEG ringing artifacts. |
| colorProfile backfill on every re-index | Migration default "color" affected all pre-v4 images. Running `refreshAllColorProfiles` after `upsertImages` (skipping the current scan batch) corrects the whole library from a single folder re-index. |
| Structured moondream prompt with Sentence 1/2 labels | Freeform prompts return empty string for ~90% of images; label scaffolding in instructions is required for reliable output even when asking moondream not to emit the labels. Retired — qwen2.5vl:7b (#20) uses a freeform paragraph prompt with no scaffolding. |
| Settings wired into EngineController at init | SettingsStore is @Observable (not ObservableObject); passed in at construction so fetchPairs can read live weights and threshold without async bridging |
| Thematic filter + live weight re-sort are display-time only | Scores stored in DB use default weights; re-sort on fetch avoids re-indexing on every settings change |
| TriangleWeightPicker takes three Binding<Double> params | Float↔Double bridging done at call site via computed Binding properties in SettingsView |
| Edge magnitude threshold raised 0.05 → 0.15 | Fine texture (brick, fabric, skin pores) was polluting the orientation histogram. Higher threshold filters these pixels before they vote, leaving only genuine structural edges. Baked-in constant in GeometricAnalyser — not a slider. |
| Geometric sub-scores stored per pair (v5) | rawEdgeSim, rawGridSim, maxEdgePeakedness, maxGridVariance persisted at score time. Enables display-time gating and re-weighting without re-score or re-index. Nil for pre-v5 pairs (fallback to stored geometricScore). |
| Display-time gating via two sliders | "Line direction strength" (edgePeakednessFloor, default 2.2) and "Composition structure floor" (gridVarianceFloor, default 0.12) apply binary discounts (40%/50%) at display time. Calibrated against library debug output: p10≈2.16 for peakedness, p10≈0.116 for variance. Floor discounts apply after the distinctiveness multiplier. |
| Distinctiveness multiplier (v6) | edgePeakednessMult = pow(normPeakA × normPeakB, 0.4); gridVarianceMult = pow(normVarA × normVarB, 0.4). Both images must be geometrically interesting for a pair to earn full credit — one flat image drags the score down proportionally. Normalization anchors at p90: kEdgePeakedNorm=4.0, kGridVarianceNorm=0.20. Exponent 0.4 (reduced from sqrt=0.5 — one calibration step toward 0 to soften the penalty). Multipliers stored per pair for display-time re-application. |
| Multiplier and floor gate are complementary, not alternatives | Multiplier: continuous, uses product of both images (both must be interesting). Floor gate: binary, uses max of both images (if even the stronger image doesn't clear the bar, discard). The multiplier fires first; the floor gate catches any remaining cases where the max is below the user's chosen threshold. |
| Orphan pair cleanup on every re-index | The per-run `DELETE FROM pairs WHERE imageAID IN (:ids) OR imageBID IN (:ids)` only covers images active in the current scan. Pairs for images deactivated via `removeFolder` were never removed, accumulating indefinitely (40,848 orphaned rows found — half the table). Added a sweep `DELETE FROM pairs WHERE imageAID NOT IN (active) OR imageBID NOT IN (active)` before the per-run DELETE. Runs unconditionally each re-index; cost is negligible on SQLite with indexed foreign keys. |
| Strip moondream label prefix at generation time | The structured prompt uses "Sentence 1:" scaffolding in its instructions; moondream (1.8B) sometimes echoes the label in output despite "No sentence labels" in rules. Previously: cleanup SQL nullified these captions, triggering a re-caption that often produced the same prefix again — an infinite cycle wasting ~152 ollama calls per run. Fix: (a) cleanup SQL now strips the prefix in-place using `TRIM(SUBSTR(caption, INSTR(caption, ': ') + 2))` instead of `SET caption = NULL`, preserving caption content; (b) `OllamaCaptioningEngine.caption()` strips the prefix from the raw response before storing. WHERE patterns unchanged (`'Sentence 1:%'` and `'First sentence:%'` only). Retired — both the cleanup SQL and the prefix-stripping code were removed when moondream was replaced by qwen2.5vl:7b (#20). |
| Weighted Dice for thematic scoring | All 24 clusters now carry a pairing weight (1.0 / 0.75 / 0.5). Score = 2×Σweights(A∩B) / (Σweights(A)+Σweights(B)). Tier 1.0: grief_sorrow, vulnerability_exposure, isolation_solitude, ritual_ceremony, tension_conflict, tenderness_care, devotion_belief, power_dominance, sensory_overwhelm. Tier 0.75: transformation_change, skilled_performance, labor_effort, stillness_rest, waiting_anticipation, movement_energy, bodily_gesture, looking_watching, confinement_freedom, youth_age, sound_music, joy_celebration. | qwen2.5vl-caption: num_ctx capped at 2048; images resized to 512px before encoding | qwen2.5vl:7b defaults to a 128k context window. Ollama allocates KV cache proportional to num_ctx at load time — 128k costs ~2GB extra RAM and dominates inference time. Full-resolution camera files (12–24MP) encode to ~2,677 prompt tokens via qwen2.5vl's tiled patch encoder, which exceeds 2048 and causes degenerate output. Fix: OllamaCaptioningEngine now resizes every image to 512px using CGImageSourceCreateThumbnailAtIndex (same pipeline as CLIP) before base64-encoding. Measured token budget at 512px: ~247 vision + ~270 prompt + 250 response cap = ~767 total. num_ctx 2048 gives ~2.5× headroom. The Modelfile lives at `ConjunctEngine/Modelfile` and registers the model as `qwen2.5vl-caption`. Do not raise num_ctx casually — even 4096 doubles the KV cache cost for no benefit at these input sizes. |
| qwen2.5vl tokenizer special-token cleanup | qwen2.5vl:7b occasionally leaks tokenizer control tokens (`<|im_start|>`, `<|im_end|>`) into the generated response when generation is cut off at the num_predict boundary. `OllamaCaptioningEngine.caption()` truncates the response at the first occurrence of `<|im_` before storing. This is preferred over a post-hoc SQL cleanup — the DB never receives a malformed caption in the first place. |

| Visual diversity multiplier on thematic score + expanded thematic topK (#45) | **Problem:** Cluster scoring rewards shared cluster volume. Same-event pairs (protest+protest, festival+festival) share many clusters → high Dice → dominate thematic topK-30. Cross-context resonant pairs (musician+ears, grief in different settings) share fewer clusters → lower Dice → excluded from topK entirely. Diagnostic confirmed: musician hero (2045) + ears hero (1523) are active, have captions, have feature vectors, but never made either image's composite topK-150 or thematic topK-30. **Root cause — topK density:** With 1,057 active images and topK=150, the 150th composite pair scores ~0.44–0.47. A cross-context pair with mismatched visual aesthetics scores below this threshold. The thematic topK-30 cutoff for the musician was ~0.46; musician+ears (estimated 0.37–0.42 thematic) fell outside. **Dead-end documented:** Caption embeddings (nomic-embed-text) tested as primary signal. Reverted: general-purpose text embeddings measure same-domain similarity, not cross-context resonance. Two festival images share festival vocabulary → high cosine → crowded out cross-context pairs further. **Fix 1 — Visual diversity multiplier (PairScorer.swift):** CLIP cosine is already computed at scoring time. Applied as a multiplier to `clusterScore` before the thematic-boost check: CLIP < 0.30 → ×1.35 (cross-context bonus); CLIP 0.30–0.60 → ×1.0 (neutral); CLIP 0.60–0.88 → ×0.75 (same-context penalty); CLIP > 0.88 handled by existing composite ceiling. **Effect:** protest+protest (CLIP ~0.65) drops from ~0.73 to ~0.55 thematic. Musician+ears (CLIP ~0.15–0.20) rises from ~0.38 to ~0.51. The multiplier applies to all pairs universally — no enumerated special cases. **Why better than complementarity bonus:** Does not require enumerating role relationships (sender/receiver). Any visually-distinct pair with cluster overlap gets rewarded regardless of the nature of the relationship. **Fix 2 — Thematic topK expansion (IndexingEngine.swift):** `thematicK = max(10, topK/5) = 30` → `max(30, topK/3) = 50` with topK=150. Applies to both Phase 1 and Phase 2 scoring. Combined with Fix 1, musician+ears thematic ~0.51 now fits within the top-50 thematic for the musician. **Pair count impact:** ~10–15% more pairs stored (~112K → ~130K expected). **Asymmetry gate confirmed not the culprit:** Ears has `isolation_solitude`/`sensory_overwhelm`/`grief_sorrow` that musician lacks; musician has `skilled_performance` that ears lacks. Both images have unique clusters; gate passes. Do not weaken or remove it. **Existing penalties preserved:** CLIP > 0.88 → composite ×0.40; CLIP > 0.75 && thematic < 0.20 → composite ×0.65. These remain unchanged — the diversity multiplier and composite ceiling are independent. |

| animal_presence demoted to ambient tier (#47) | **Diagnostic (2026-05-06, post-#45 re-index):** Dog+dog pairs ranked #7 globally (images 1447+1624, composite 0.623, thematic 0.547). Multiple other animal-mention pairs in the 0.58–0.62 composite range. The `animal_presence` cluster at weight 0.75 was the cause — it gave "two dogs" a meaningful thematic signal, exactly the same-same pattern the system is supposed to reduce.
